# Agent Activity Bus

A personal dashboard that tracks what your AI development agents are doing across all your GitHub repos.

## Overview

When you're running agentic development across multiple projects and tools simultaneously, it becomes hard to remember where you left off, what's waiting for your attention, and what you should work on next. This tool answers those questions in under 30 seconds.

## How It Works

1. **GitHub webhooks** capture commits, pushes, and PR merges across your repos
2. **Commit messages** become activity updates (already meaningful)
3. **AI summarizes** project status on demand (cheap, fast model using your API keys)
4. **Dashboard** shows what needs your attention, with merged branches clearly marked

No per-project setup. One webhook configuration covers all repos.

## Features

- **Activity Feed** - Commit-based updates from all your repos
- **Project Pages** - Story of where each project is right now, with merged branches visually distinguished
- **AI Summaries** - On-demand feature-level summaries from recent activity (Anthropic/OpenAI/Google)
- **Action Tags** - Auto-generated indicators (needs_attention, review_requested, ready_to_merge, etc.)
- **Work Classification** - Automatic categorization (feature/bugfix/refactor/infrastructure/docs/maintenance)
- **Merge Detection** - PR webhooks mark completed work, sorted after active branches
- **Global Pause** - Disable tracking when needed

## Setup

### 1. Deploy the Dashboard

```bash
npm install
cd functions && npm install && cd ..
firebase deploy
```

### 2. Create an Agent Token

1. Sign in to the dashboard
2. Go to Settings → Agent Tokens
3. Create a new token

### 3. Configure GitHub Webhook

**Option A: GitHub App (recommended for all repos):**

Create a GitHub App at https://github.com/settings/apps/new:
- **Webhook URL**: `https://us-central1-town-center-agent.cloudfunctions.net/githubWebhook?token=YOUR_TOKEN`
- **Permissions**: Contents (read), Metadata (read), Pull requests (read)
- **Events**: Push, Pull request
- Install on your account to auto-track all repos

**Option B: Manual webhook (single repo):**

1. Go to repo → Settings → Webhooks
2. Add webhook:
   - **Payload URL**: `https://us-central1-town-center-agent.cloudfunctions.net/githubWebhook?token=YOUR_TOKEN`
   - **Content type**: `application/json`
   - **Events**: `push` and `pull_request`
3. Save

That's it. All commits and PR merges will now appear in your dashboard.

## Data Architecture

```
Stored in Firestore:
├── Activity updates (from GitHub webhooks)
├── Branch/workstream metadata (status, merge info)
├── User preferences
└── Token metadata

Generated by AI (on demand):
├── Feature-level summaries
├── Action tags
├── Work type classification
└── Project status summaries
```

**Philosophy**: Store minimal data. Let GitHub be the source of truth. Use AI to synthesize meaning on demand.
