# Agent Activity Bus

A personal dashboard that tracks what your AI development agents are doing across all your GitHub repos.

## Overview

When you're running agentic development across multiple projects and tools simultaneously, it becomes hard to remember where you left off, what's waiting for your attention, and what you should work on next. This tool answers those questions in under 30 seconds.

## How It Works

1. **GitHub webhooks** capture all commits/pushes across your repos
2. **Commit messages** become activity updates (already meaningful)
3. **AI summarizes** project status on demand (cheap, fast model)
4. **Dashboard** shows what needs your attention

No per-project setup. One webhook configuration covers all repos.

## Features

- **Activity Feed** - Commit-based updates from all your repos
- **Project Pages** - Story of where each project is right now
- **AI Summaries** - On-demand status synthesis from recent activity
- **Global Pause** - Disable tracking when needed

## Tech Stack

- React 19 + TypeScript + Vite
- Firebase (Auth, Firestore, Cloud Functions, Hosting)
- TailwindCSS 4

## Setup

### 1. Deploy the Dashboard

```bash
npm install
cd functions && npm install && cd ..
firebase deploy
```

### 2. Create an Agent Token

1. Sign in to the dashboard
2. Go to Settings → Agent Tokens
3. Create a new token

### 3. Configure GitHub Webhook

**For all repos (recommended):**

1. Go to GitHub → Settings → Webhooks (account level)
2. Add webhook:
   - **Payload URL**: `https://us-central1-town-center-agent.cloudfunctions.net/githubWebhook?token=YOUR_TOKEN`
   - **Content type**: `application/json`
   - **Events**: Just the `push` event
3. Save

**For a single repo:**

Same steps, but in the repo's Settings → Webhooks.

That's it. All commits will now appear in your dashboard.

## Data Architecture

```
Stored in Firestore:
├── Activity updates (from GitHub webhooks)
├── User preferences
└── Token metadata

Fetched from GitHub (on demand):
├── Recent commits for context
├── Open PRs
└── Branch info

Generated by AI (on demand):
├── Project status summaries
└── Workstream narratives
```

**Philosophy**: Store minimal data. Let GitHub be the source of truth. Use AI to synthesize meaning on demand.
